# Web Platform Guide

**Last verified:** 2026-03-27

## Overview

This document covers web-specific development considerations for Flutter web apps targeting modern browsers.

## Browser Support

| Browser | Version | Support Status |
|---------|---------|----------------|
| Chrome | 90+ | Primary target |
| Edge | 90+ | Fully supported |
| Safari | 14+ | Fully supported |
| Firefox | 90+ | Fully supported |

Set minimum versions in `web/index.html` meta tags.

## Web-Specific Flutter

### Setup
```bash
flutter create --platforms=web
flutter build web
```

### Key Differences
- No native plugins for camera, location without web equivalents
- Use `url_launcher` for opening URLs
- Use `universal_html` for web APIs
- Web builds to `build/web/` directory

## PWA (Progressive Web App)

### Manifest
Configure in `web/manifest.json`:
```json
{
  "name": "My App",
  "short_name": "App",
  "start_url": ".",
  "display": "standalone",
  "background_color": "#FFFFFF",
  "theme_color": "#000000",
  "icons": [...]
}
```

### Service Worker
- Generated by Flutter in release builds
- Enable offline support
- Cache strategies: cache-first, network-first

### Installation
- Add to home screen prompt support
- Web app manifest required
- HTTPS required for PWA features

## Responsive Design

### Breakpoints
```dart
enum ScreenSize { mobile, tablet, desktop }

ScreenSize getScreenSize(BuildContext context) {
  final width = MediaQuery.of(context).size.width;
  if (width < 600) return ScreenSize.mobile;
  if (width < 900) return ScreenSize.tablet;
  return ScreenSize.desktop;
}
```

### Layout Patterns
- Mobile: Single column
- Tablet: Two column where appropriate
- Desktop: Full multi-column layouts

### Viewport
Set in `web/index.html`:
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no">
```

## URL Strategy

### Hash Strategy (default)
```
example.com/#/settings
```

### Path Strategy
```dart
// In MaterialApp
urlStrategy: PathUrlStrategy()
```
Requires server configuration for deep links.

### Server Configuration (Path Strategy)

**Nginx:**
```nginx
location / {
  try_files $uri $uri/ /index.html;
}
```

**Firebase Hosting:**
```json
{
  "hosting": {
    "public": "build/web",
    "rewrites": [{"source": "**", "destination": "/index.html"}]
  }
}
```

## Web-Specific Issues

### CanvasKit vs HTML Renderer
- CanvasKit: Better graphics, larger bundle (~2MB)
- HTML: Smaller bundle, better text rendering

```bash
flutter build web --web-renderer html  # Smaller
flutter build web --web-renderer canvaskit  # Better graphics
```

### Keyboard Events
```dart
Focus(
  onKeyEvent: (node, event) {
    // Handle keyboard events
  },
  child: ...,
)
```

### Clipboard
```dart
import 'package:flutter/services.dart';
await Clipboard.setData(ClipboardData(text: 'Hello'));
```

### File Downloads
```dart
import 'dart:html' as html;
final blob = html.Blob([data]);
html.AnchorElement(href: html.Url.createObjectUrlFromBlob(blob))
  ..setAttribute('download', 'file.txt')
  ..click();
```

## Performance

### Optimization
- Use `const` constructors
- Lazy load heavy widgets
- Minimize rebuilds with `RepaintBoundary`
- Enable tree shaking

### Bundle Size
- Target: <2MB initial load
- Use `flutter build web --release`
- Enable deferred loading

### Core Web Vitals
- LCP (Largest Contentful Paint): <2.5s
- FID (First Input Delay): <100ms
- CLS (Cumulative Layout Shift): <0.1

## Accessibility

- Semantic widgets required
- Keyboard navigation support
- ARIA labels
- Screen reader testing (NVDA, VoiceOver)

## Testing

- Chrome DevTools primary
- Responsive testing in DevTools
- Lighthouse audits
- Cross-browser testing
